aca un ejemplo de como conectarse a oracle:

En este proyecto, la conexión a Oracle se realiza utilizando Python con las librerías **`jaydebeapi`** y **`JPype1`**, utilizando un driver JDBC local de Oracle (por defecto `jdbc/ojdbc8 copy.jar`). 

Una característica arquitectónica fundamental es que el sistema usa un **patrón de conexión con "failover" manual** para entornos RAC. Esto significa que el código itera sobre una lista de nodos de base de datos disponibles y prueba conectarse a ellos uno por uno hasta lograr una conexión exitosa.

A continuación, te muestro el ejemplo de cómo se implementa internamente esta conexión (basado en la función `_cd_oracle_connect` del código del sistema):

```python
import os
from pathlib import Path
import jaydebeapi

def _cd_oracle_connect():
    # 1. Obtener y validar la ruta del driver JDBC local
    jar = Path(os.environ.get("ORACLE_JDBC_JAR", "jdbc/ojdbc8 copy.jar")).expanduser()
    if not jar.exists():
        raise RuntimeError(f"Jar not found: {jar}")

    # 2. Cargar credenciales desde variables de entorno (usualmente del .env)
    user = os.environ.get("ORACLE_USER", "")
    password = os.environ.get("ORACLE_PASSWORD", "")
    if not user or not password:
        raise RuntimeError("ORACLE_USER or ORACLE_PASSWORD not set")

    # 3. Obtener la lista de nodos (targets) para failover
    # El formato que devuelve es una lista de tuplas: (host, puerto, sid)
    targets = _oracle_targets_from_env() 
    driver = "oracle.jdbc.OracleDriver"

    # 4. Intentar conectarse a cada nodo en orden
    for host, port, sid in targets:
        url = f"jdbc:oracle:thin:@{host}:{port}:{sid}"
        try:
            # Conexión ejecutada mediante JayDeBeApi enviando el string de conexión JDBC
            return jaydebeapi.connect(driver, url, [user, password], jars=[str(jar)])
        except Exception:
            continue # Si el nodo falla, intenta con el siguiente en la lista

    # Si se agota la lista y ninguno respondió, se levanta una excepción
    raise RuntimeError("Failed to connect to Oracle")
```

**Puntos vitales de configuración para esta conexión:**

1. **Variables de entorno obligatorias:** El código espera que estén definidas `ORACLE_USER`, `ORACLE_PASSWORD`, `ORACLE_TARGETS` (ej: `172.16.60.20:1521:prdsgh1,172.16.60.21:1521:prdsgh2`), y opcionalmente `ORACLE_JDBC_JAR`.
2. **Prevención del error `ORA-01882` (Workaround):** Debido a discrepancias de zona horaria entre Java y el servidor Oracle, es obligatorio que el proceso o servicio de Python arranque con la siguiente variable seteada para prevenir caídas de conexión: 
   ```bash
   JAVA_TOOL_OPTIONS="-Doracle.jdbc.timezoneAsRegion=false -Duser.timezone=UTC"
   ```
